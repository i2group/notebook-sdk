    Writing records, changing selection and view

    As described in the Changing data with mutations guide i2 Notebook Web API provides the ability to write data to the chart and change application state.

    The types of changes that can be made are:

    • Record creation
    • Record editing
    • Record removal
    • Selection changes
    • View changes
    • Visual node element position

    A single mutation action can contain any number and combination of those changes. The changes are typically applied as a single set of changes once the mutation itself is committed.

    Some of these commands are demonstrated in the Mutations Plug-in tutorial.

    Record creation and editing

    The a mutation handler supplied to a ('ITrackedMutations.runTrackedMutation')(../api/notebook-sdk.app.iapplication.runtrackedmutations.md) allows the creation of new entity or link records, via the ITrackedMutations.addEntityRecord and ITrackedMutations.addLinkRecord methods. Each call allows the features of the new record to be supplied, including the item type, notes, security settings and values for properties.

    Both methods return a "pending" record object that represent the new record before it has been added to the chart data, this object implements the IRecordEditor interface that allow further edits to be made.

    It is also possible to make changes to existing records on the chart by using the ITrackedMutations.editRecord method, that will return a IRecordEditor interface for that record.

    Note: New records will not exist on the chart, and therefore will not be seen in IChart collections, until the mutation is committed. Pending records and record editors are only valid within the mutation handler where they were created.

    Value factories

    When creating or editing record you are able to supply a set of property values to apply to the new record. To assist with creating valid value object, the ITrackedMutations.valueFactory property supplies a IValueFactory interface that contains a number of helper methods.

    These methods will assist you in creating valid object that implement the following interfaces:

    • IGeopoint
    • IDecimal
    • ILocalDate
    • ILocalDateTime
    • ILocalTime
    • ISecuritySettings
    • IZonedDateTime

    Record and element removal

    As well as creating new records, it is also possible to remove them via the ITrackedMutations.removeRecords method.

    You may also remove the visual nodes and edge elements from the chart by calling the ITrackedMutations.removeElements method.

    Any records held by a visual element will also be removed, together with any connected records and elements.

    Changing selection

    Mutations may change the selection state of the chart via the ISelectionEditor interface. Both tracked and untracked mutations allow the changing of selection, via IMutationsBase.selection.

    Note: While selection changes made in untracked actions are not explicitly added to the undo stack, the selection state can be captured by the next tracked mutation as part of it's undo-able state.

    Moving visual node elements

    It is possible to arrange visual node elements on the chart by changing their position:

    • By setting a single node position directly using the ITrackedMutations.setNodeCenter method method.
    • By moving a selection that includes node elements by an offset using the ITrackedMutations.moveSelectedNodesRelative method

    Changing the view

    Finally it's possible to change the current view of the chart, moving the where the view is centered and the zoom level.

    These changes are made by calling methods on the IViewEditor interface, accessed by the IMutationsBase.view property.

    Validation

    All the mutation command will validate their parameters to ensure the correct value types are passed and that when record and type ids are given they are associated with existing objects.
