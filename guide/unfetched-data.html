    Unfetched property values

    In order to reduce memory and bandwidth consumption, i2 Notebook does not always fetch every property value for every record into the web client.

    It is possible to distinguish between a property that has no value and a property whose value has not yet been fetched using the isValueUnfetched method on an individual property value, or by reading hasUnfetchedProperties or calling getUnfetchedPropertyTypes on a record.

    Requesting unfetched property values

    It is possible to request unfetched property values within a transaction or mutation by calling the IChart.ensurePropertiesFetched method.

    This method will fetch the specified property types for the specified records, from the server, asynchronously.

    For example:

    api.runTransaction(async (application) => {
  const record = application.chart.records.get(myRecordId);
  const propertyType = record.itemType.propertyTypes.get(myPropertyTypeId);

  // Check if the property value of type `propertyType` has been fetched on `record`
  // and if not, request the data for that property value from the server.
  await application.chart.ensurePropertiesFetched([record], propertyType);
});

    Note that, for performance, ensurePropertiesFetched deliberately takes an array of records, and will fetch properties for many records efficiently in a small number of server round-trips. Calling it repeatedly with a single record at a time will produce much poorer performance and increased network usage.

    The fetching of property values is not considered a "mutation" and will not be part of any tracked mutation undo action.

    Note: It is possible that property values cached by ensurePropertiesFetched may be 'evicted' from the client memory, so you should not rely upon fetched values remaining fetched.
