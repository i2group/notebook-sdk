    Creating a plugin to write changes to the i2 Notebook web client chart

    This tutorial takes you through writing a plug-in for the i2 Notebook web client that can write new records to the chart, arranging them and update the view.

    You can find a full version of the source code for this tutorial in the samples/mutation-plugin folder.

    Prerequisities

    This tutorial requires a running instance of i2 Analyze on which you have permission to use the i2 Notebook web client.

    To follow the tutorial, you must install at least version 12 of Node.js on your development machine. For downloads and more information about Node.js, visit the project website at https://nodejs.org.

    The tutorial also requires you to create GUIDs for a number of its elements. The project files use some sample fixed GUIDs (00000000-0000-0000-0000-000000000001, 00000000-0000-0000-0000-000000000002, and so on), but you should always create real GUIDs for real projects.

    Create a basic plug-in

    To start with we'll ask you to follow the basic plug-in tutorial, up to but not including the step where we add the first Ribbon command. Make sure you have a working plugin before continuing.

    Add a ribbon command

    Now we have an empty plugin, we'll start by adding a command to the ribbon that will add some Person records to the chart.

    1. In entrypoint.js, just before the call to initializationComplete(), create a command and surface the command on the ribbon.

      const addItems = api.commands.createCommand({
  id: '00000000-0000-0000-0000-000000000002',
  name: 'Add items',
  type: 'unscoped',
  icon: {
    type: 'inlineSvg',
    svg: '<svg viewBox="0 0 16 16"><rect width="8" height="8" x="4" y="4"/></svg>',
  },
  onExecute() {},
});

api.commands.applicationRibbon.homeTab
  .after(api.commands.applicationRibbon.homeTab.systemGroups.searchInfoStore)
  .surfaceCommands(addItems);

    2. Reload i2 Notebook and check that the action is visible in the application ribbon.

    Add entity records

    1. At the top of the file, add some dummy data that will serve as the source of the items that we'll add to the chart.

      const personData = [
  {
    firstName: 'Lou',
    familyName: 'Tuft',
  },
  {
    firstName: 'Jessy',
    familyName: 'House',
  },
  {
    firstName: 'Jayme',
    familyName: 'Tiffy',
  },
  {
    firstName: 'Lauren',
    familyName: 'Timberlake',
  },
  {
    firstName: 'Romilly',
    familyName: 'Chaplin',
  },
];

    2. Also define the entity type and property types from the schema that will be use to create the entities and links, place this at the top of the file next to the dummy data.

      const personEntityTypeId = 'ET5';
const firstNamePropertyTypeId = 'PER4';
const familyNamePropertyTypeId = 'PER6';

    3. Next to this add the following helper functions that allows us to look up an item types and property types by an analyze id.

      /**
 * @param {import("@i2analyze/notebook-sdk").schema.IChartSchema} schema
 * @param {string} analyzeId
 * @returns {import("@i2analyze/notebook-sdk").schema.IChartItemType}
 */
function getItemType(schema, analyzeId) {
  const itemType = schema.itemTypes.find((v) => v.analyzeId === analyzeId);
  if (!itemType) {
    throw new Error(`Schema is not as expected: ${analyzeId} is missing`);
  }
  return itemType;
}

/**
 * @param {import("@i2analyze/notebook-sdk").schema.IChartItemType} itemType
 * @param {string} analyzeId
 * @returns {import("@i2analyze/notebook-sdk").schema.IChartPropertyType}
 */
function getPropertyType(itemType, analyzeId) {
  const propertyType = itemType.propertyTypes.find((v) => v.analyzeId === analyzeId);
  if (!propertyType) {
    throw new Error(`Schema is not as expected: ${analyzeId} is missing`);
  }
  return propertyType;
}

      Note: The /** @type {import...} */ and /** @param {import...} */ code helps the JavaScript type system to understand what we're working with, which enables auto-completion in the rest of the code. If we used TypeScript to create our entrypoint.js file, we'd use import type statements instead.

    4. Inside the implementation of onExecute add the following code to get the entity and property types that are needed:

      const personEntityType = getItemType(application.chart, personEntityTypeId);
const firstNamePropertyType = getPropertyType(personEntityType, firstNamePropertyTypeId);
const familyNamePropertyType = getPropertyType(personEntityType, familyNamePropertyTypeId);

    5. Immediately after add a call to runTrackedMutation to start queuing up mutation commands that will add some items to the chart.

      api.runTrackedMutations((_application, mutations) => {
  // Mutations go here
});

    6. Now we want to start adding mutation commands inside the runTrackedMutations call, so let's loop over the items in our data object, adding them as entities on the chart by using our entity type and property types we defined earlier. Add this code inside the function we pass to runTrackedMutations.

      for (const person of personData) {
  mutations.addEntityRecord({
    itemType: personEntityType,
    properties: {
      [firstNamePropertyType]: person.firstName,
      [familyNamePropertyType]: person.familyName,
    },
  });
}

      Alternatively we could add the entity record first, and then "edit" the new record:

      for (const person of personData) {
  const newPerson = mutations.addEntityRecord({
    itemType: personEntityType,
  });

  newPerson.setProperties({
    [firstNamePropertyType]: person.firstName,
    [familyNamePropertyType]: person.familyName,
  });
}

    7. To complete the runTrackedMutation call we must return an object that tells the application whether to commit the mutation commands. Since this is a tracked mutation we must also supply the name of the undo entry that will be added to the undo stack. Add this just after the for loop.

      return {
  type: 'commit',
  actionDisplayName: 'Add items',
};

      A tracked mutation will add a entry to the undo stack. If that Undo action is executed, all the mutations commands that were added will be reversed, leaving the chart back in the state before the runTrackedMutations call.

      An untracked mutation will not add an entry to the undo stack, but the mutation commands that can be added are limited to selection and viewport mutation. Those changes will become part of the next tracked mutation and will be undone as part of that mutation.

    8. Reload i2 Notebook and try running the command. You should see a number of Person records be added to the chart, which are contained within node elements on the visible chart surface. Next we'll add some more records that we'll link to the these Person records.

    Add linked records

    1. Add another set of data we'll use to define some events, with a "wasSeen" property that determines which person record the event should be linked to.

      const eventEntityTypeId = 'ET2';
const eventTypePropertyTypeId = 'EVE3';
const reportedDateTimePropertyTypeId = 'EVE34';
const observedLinkTypeId = 'LOB1';

const eventData = [
  {
    date: '2022-03-01',
    wasSeen: 'Tuft',
  },
  {
    firstName: '2021-11-23',
    wasSeen: 'House',
  },
  {
    firstName: '2022-05-07',
    wasSeen: 'Tiffy',
  },
];

    2. Add some more entity, link and property types, placing these just after the existing type objects we defined previously:

      const eventEntityType = getItemType(application.chart, eventEntityTypeId);
const eventTypePropertyType = getPropertyType(eventEntityType, eventTypePropertyTypeId);
const startDateTimePropertyType = getPropertyType(eventEntityType, startDateTimePropertyTypeId);
const observedLinkType = getItemType(application.chart, observedLinkTypeId);

    3. For the event timezones, we'll add another helper function to look up the timezones by their id. Add this after our other helper functions at the top of the file:

      ```javascript
/**
 * @param {import("@i2analyze/notebook-sdk").data.IKeyedReadOnlyCollection<string,import("@i2analyze/notebook-sdk").data.ITimeZone>} timeZones
 * @param {string} timeZoneId
 */
function getTimeZone(timeZones, timeZoneId) {
  const timeZone = timeZones.get(timeZoneId);
  if (timeZone === undefined) {
    throw new Error('Unknown time zone matching ${timeZoneId}');
  }
  return timeZone;
}
```

    4. To help find the person record to which we want to link the event records, we'll add a Map object into which we'll add our person record, keyed by their family name value.

      Note: Since the records aren't added to the chart until the mutation is actually committed and run, we can't simply look them up on the chart yet.

    5. Replace the for loop where we add the person records with this code, which adds the new records to the look-up Map.

      /**
 * @type {Map<string,import("@i2analyze/notebook-sdk").app.IPendingRecord>}
 */
const personLookup = new Map();

for (const person of personData) {
  const personRecord = mutations.addEntityRecord({
    itemType: personEntityType,
    properties: {
      [firstNamePropertyType.id]: person.firstName,
      [familyNamePropertyType.id]: person.familyName,
    },
  });

  personLookup.set(person.familyName, personRecord);
}

    6. Now we'll use our new data to create more records. Firstly we'll add some events, in the runTrackedMutation call, just before we return from the mutation:

      for (const event of eventData) {
  const startDateTime = mutations.valueFactory.createZonedDateTime(
    event.dateTime,
    getTimeZone(event.timeZone),
    false
  );

  const eventRecord = mutations.addEntityRecord({
    itemType: eventEntityType,
    properties: {
      [eventTypePropertyType.id]: event.type,
      [startDateTimePropertyType]: startDateTime,
    },
  });
}

      Note: The mutations object provides a number of value factory methods that we can use to create validated values to use when setting properties. We will use the createZonedDateTime method to set the event reported date-time property:

    7. Next we'll add a link after adding each Event entity, using the lookup map to find the Person record on the other end. Add this inside the loop, just after creating the event record.

      const personSeenRecord = personLookup.get(event.wasSeen);

if (personSeenRecord === undefined) {
  throw new Error(`Person matching ${event.wasSeen} is missing`);
}

mutations.addLinkRecord({
  itemType: observedLinkType,
  fromEnd: eventRecord,
  toEnd: personSeenRecord,
  linkDirection: 'with',
});

    8. Reload i2 Notebook delete everything from the chart and run the command. You should see the Person records being added, each one linked to an Event record.

    Mutating Selection

    Now have the events added to the chart, let's also add them to the selection.

    1. Add this just after adding the link record:

      mutations.selection.add(eventRecord);

    2. Reload i2 Notebook again, this time when you run the command you should see that all the Event records are now selected.

    Mutating Elements and Mutation Roll-back

    As well as adding records, we can mutate the visual elements that have been created, let's try arranging our selected events in a left-to-right time sequence:

    1. Create a new command and surface it in the ribbon:

      const arrangeEvents = api.commands.createCommand({
  id: '00000000-0000-0000-0000-000000000003',
  name: 'Arrange events',
  type: 'unscoped',
  icon: {
    type: 'inlineSvg',
    svg: '<svg viewBox="0 0 16 16"><rect width="8" height="8" x="4" y="4"/></svg>',
  },
  onExecute(application) {
    // Execute code goes here
  },
});

api.commands.applicationRibbon.homeTab.after(addItems).surfaceCommands(arrangeEvents);

    2. Before we arrange the Event elements, let's first validate that the current selection only contains event type records, if not we'll "rollback" the mutation and show a notification of the problem:

      api.runTrackedMutations((application, mutations) => {
  const selection = application.chart.selection;

  if (
    !selection.entityRecords.every((record) => record.itemType.analyzeId === eventEntityTypeId)
  ) {
    return {
      type: 'rollback',
      report: {
        details: 'Selection must contain only Event record types',
        title: 'Cannot arrange items',
        type: 'error',
      },
    };
  }

  return {
    type: 'commit',
    actionDisplayName: 'Arrange events',
  };
});

      If you now select all of the items that have been previously selected and try using the Arrange Items command, an error notification will appear, since the selection contains more than just events.

      We've just rolled back a mutation that actually contains no mutation commands, that's a valid operation can be used to display a notification. However if you try to "commit" a mutation that has no mutation commands a error will be shown, since it's not valid to try and commit an empty mutation You'll see this happen right if you have a "valid" selection, since we're not yet running any mutation commands after we've checked the selection.

    Now we can start a new mutation and use the current selection to find events and arrange them in a sorted-by-date-time order.

    1. Add the following code after our selection check, this sorts the selected events and uses setNodeCenter to move them into their new positions.

      const eventEntityType = getItemType(application.chart, eventEntityTypeId);
const reportedDateTimePropertyType = getPropertyType(
  eventEntityType,
  reportedDateTimePropertyTypeId
);

/**
 * @param {import("@i2analyze/notebook-sdk").visual.INode} node
 */
function getEventDate(node) {
  const eventRecord = node.records.firstOrDefault(undefined);

  if (eventRecord === undefined) {
    throw new Error('Unexpected missing record');
  }

  const eventDateTime = eventRecord.getProperty(reportedDateTimePropertyType);

  if (eventDateTime !== undefined && !eventRecord.isValueUnfetched(eventDateTime)) {
    return /** @type {import("@i2analyze/notebook-sdk").data.IZonedDateTime} */ (eventDateTime);
  }
}

/**
 * @param {import("@i2analyze/notebook-sdk").visual.INode} nodeA
 * @param {import("@i2analyze/notebook-sdk").visual.INode} nodeB
 */
function compareEvents(nodeA, nodeB) {
  const eventADate = getEventDate(nodeA);
  const eventBDate = getEventDate(nodeB);

  if (eventADate !== undefined && eventBDate !== undefined) {
    const dateA = eventADate.dateTime.toJSDate().getTime();
    const dateB = eventBDate.dateTime.toJSDate().getTime();
    return dateA - dateB;
  } else {
    return 0;
  }
}

const firstEvent = selection.affectedNodes.firstOrDefault(undefined);
if (!firstEvent) {
  return {
    type: 'rollback',
    report: {
      title: 'Nothing selected',
      details: 'Nothing was selected, so no events were arranged.',
      type: 'information',
    },
  };
}

const sortedEvents = Array.from(selection.affectedNodes).sort(compareEvents);

const targetLocation = firstEvent.center;
let targetX = targetLocation.x;

for (const event of sortedEvents) {
  mutations.setNodeCenter(event, { x: targetX, y: targetLocation.y });
  targetX += 200;
}

    2. Lastly we can mutate the view to fit the selected events, add this just before we return from the mutation call:

      mutations.view.fitToSelection();

    3. Reload i2 Notebook and run the command, you should see that the selected elements have been moved into a left-to-right temporal layout.
